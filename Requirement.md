# 智读——AI分层阅读器项目需求说明书

## 一、项目概述

### 1.1 核心理念

通过AI分层解析，实现"先读薄，再读厚"的阅读体验，最终将书籍精华沉淀为可复习的个人知识资产。本产品定位于个人知识管理工具，而非复杂的社交或游戏化平台。

### 1.2 产品定位

一款基于Flutter开发的跨平台AI智能阅读器，主打"分层阅读"和"知识蒸馏"功能。

## 二、核心功能模块

### 2.1 智能导入与解析

**支持格式**：EPUB、PDF

**解析策略**：
- 采用"精准打击"原则，避免全量解析
- EPUB：利用EPUB自带的章节标签和导航结构，优先从OPF/Toc识别章节结构
- PDF：智能识别章节标题（支持中文/英文章节编号格式），自动跳过封面页

**技术实现**：
- EPUB：基于epub_plus库进行EPUB解析
- PDF：基于pdf库进行PDF渲染，使用sync_pdf_renderer进行分页处理

### 2.2 AI分层阅读引擎

**全书概览生成逻辑**：
- **有前言/序言**：直接从前言内容生成全书摘要（800-900字，Markdown格式）
- **无前言/序言**：等待所有章节摘要生成完成后，基于章节摘要生成全书摘要

**章节精炼**：用户点击目录进入某一章时，AI对该章节内容进行摘要提炼（800-900字，Markdown格式）

**输出格式**：统一使用Markdown格式，包含标题、段落、列表等，便于阅读和导出

### 2.3 用户交互与控制

**逃逸机制**：在任何AI摘要页面提供"阅读原文"按钮，直接跳转到原文

**智能预加载**：在用户打开书籍详情时，后台静默预生成章节摘要

**章节导航**：仅在第一级目录间遍历，避免子章节干扰用户体验

**PDF阅读**：分页显示PDF内容，自动跳过封面页（文本少于50字符的首页）

## 三、数据存储与架构

### 3.1 存储方案

**文件存储**：采用JSON + Markdown文件存储方案，替代原有SQLite数据库

**优势**：
- 更易于数据迁移和备份
- 直接输出Markdown格式，无需额外转换
- 简化数据模型，降低维护复杂度

### 3.2 文件结构

数据存储在应用专用目录 `Documents/zhidu/`：

```
Documents/zhidu/
├── books.json          # 书籍索引文件
└── books/
    └── {bookId}/       # 每本书独立目录
        ├── metadata.json      # 书籍元数据
        ├── summary.md         # 书籍摘要
        ├── chapter-001.md     # 章节摘要（按章节编号）
        ├── chapter-002.md
        └── cover.jpg/png      # 封面图片
```

### 3.3 存储路径

- **Windows**: `C:\Users\{username}\Documents\zhidu\`
- **macOS**: `/Users/{username}/Documents/zhidu/`
- **Android**: `/storage/emulated/0/Documents/zhidu/` 或应用私有目录
- **iOS**: `/var/mobile/Containers/Data/Application/{uuid}/Documents/zhidu/`

### 3.4 读写逻辑

**写入逻辑**：
- 书籍元数据保存为JSON文件 (`metadata.json`)
- 章节摘要保存为Markdown文件 (`chapter-{index}.md`)
- 全书摘要保存为Markdown文件 (`summary.md`)

**读取逻辑**：
- 从JSON文件读取书籍列表和元数据
- 从Markdown文件读取摘要，使用flutter_html渲染

**导出逻辑**：支持将所有摘要打包导出为Markdown文件

## 四、技术实现路径

### 4.1 技术栈

**前端框架**：Flutter（Dart）

**文件处理库**：
- EPUB：epub_plus、archive、xml、image
- PDF：pdf、sync_pdf_renderer
- 文件：file_picker、path_provider、path

**AI服务**：集成智谱/通义千问大语言模型API

**存储方案**：文件存储（JSON + Markdown）

**UI渲染**：flutter_html（Markdown/HTML渲染）

**Phase 3：功能扩展（待开发）**

## 五、UI设计说明

### 5.1 整体设计原则

- **简洁直观**：界面设计简洁明了，突出核心功能
- **层次分明**：通过视觉层次区分不同内容模块
- **交互友好**：提供直观的交互方式，降低用户学习成本

### 5.2 主要界面设计

#### 5.2.1 首页/书架界面

**布局**：
- 顶部：应用标题"智读"
- 中部：书籍列表，采用卡片式布局
- 底部：导航栏（书架、发现、设置）

**功能元素**：
- 书籍卡片包含：封面图、书名、作者、阅读进度
- 添加书籍按钮（浮动操作按钮）

#### 5.2.2 书籍详情界面

**布局**：
- 顶部：书籍封面和基本信息
- 中部：目录结构（可折叠）
- 底部：全书概览（Markdown格式）

**功能元素**：
- 点击章节进入阅读
- 全书概览展示区（Markdown渲染）

#### 5.2.3 章节摘要界面

**布局**：
- 顶部：章节标题和导航
- 中部：AI生成的章节摘要（Markdown格式）
- 底部：原文链接和章节导航按钮

**功能元素**：
- "阅读原文"按钮
- `<` 和 `>` 导航按钮（仅在第一级章节间遍历）
- Markdown格式的摘要内容

#### 5.2.4 PDF阅读界面

**布局**：
- 顶部：章节/页面标题
- 中部：PDF页面渲染
- 底部：页面导航控制

**功能元素**：
- 分页显示PDF内容
- 自动跳过封面页
- 页面导航按钮

### 5.3 视觉设计规范

#### 5.3.1 色彩方案

- **主色调**：深蓝色（#2C3E50）用于标题和重要元素
- **辅助色**：橙色（#E67E22）用于强调元素
- **背景色**：浅灰色（#F8F9FA）用于页面背景
- **卡片色**：白色（#FFFFFF）用于内容卡片

#### 5.3.2 字体规范

- **标题字体**：系统默认Bold
- **正文字体**：系统默认Regular

#### 5.3.3 图标设计

- 采用Material Design风格图标
- 统一的图标尺寸和线条粗细
- 清晰的视觉识别度

## 六、非功能性需求

### 6.1 性能需求

**响应速度**：AI摘要生成的等待时间应控制在用户可接受范围内

**资源占用**：合理管理内存，避免因大文件解析导致的卡顿或崩溃

### 6.2 安全与隐私

**本地存储**：所有书籍文件与数据均存储在用户设备本地

**API密钥保护**：AI配置文件（ai_config.json）已加入.gitignore，不会泄露到版本控制

### 6.3 兼容性需求

**平台兼容**：支持Windows（主要开发）、Android、iOS、macOS、Linux

**Web限制**：Web平台仅支持演示，无法导入本地文件（浏览器沙箱限制）

**文件格式**：兼容主流EPUB和PDF格式

## 八、风险评估与应对

### 8.1 技术风险

**EPUB/PDF解析复杂性**：不同格式可能导致解析困难
**应对措施**：使用成熟的解析库，提供回退解析机制

### 8.2 AI服务风险

**API成本控制**：AI调用可能产生费用
**应对措施**：提供AI配置选项，支持用户选择不同提供商

### 8.3 用户体验风险

**Markdown渲染一致性**：不同平台Markdown渲染可能有差异
**应对措施**：使用flutter_html确保渲染一致性
