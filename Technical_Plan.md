# 智读——AI分层阅读器项目技术方案

## 一、项目概述

### 核心理念
通过AI分层解析，实现"先读薄，再读厚"的阅读体验，最终将书籍精华沉淀为可复习的个人知识资产。本产品定位于个人知识管理工具，而非复杂的社交或游戏化平台。

### 产品定位
一款基于Flutter开发的跨平台AI智能阅读器，主打"分层阅读"和"知识蒸馏"功能。

## 二、技术架构

### 2.1 技术栈
- **前端框架**：Flutter（Dart）
- **文件处理库**：epub_plus（EPUB章节解析）、archive（文件解压）、xml、image
- **AI服务**：集成智谱/通义千问大语言模型API
- **存储方案**：SQLite数据库（drift ORM）
- **状态管理**：StatefulWidget + Service单例模式
- **UI渲染**：flutter_html（Markdown/HTML渲染）

### 2.2 模块划分
1. **核心阅读模块**：负责EPUB文件的导入、解析和渲染
2. **AI分析模块**：负责调用AI服务生成章节摘要和全书概览
3. **数据存储模块**：负责使用SQLite数据库存储书籍信息和摘要
4. **导出模块**：负责将摘要导出为Markdown格式
5. **UI/UX模块**：负责所有界面的设计和交互实现

### 2.3 数据流设计
```
文件导入 → 解析（EPUB） → AI分析（章节摘要） → 展示（Markdown格式） → 存储（SQLite数据库） → 导出（Markdown文件）
```

## 三、核心功能实现方案

### 3.1 智能导入与解析
- **支持格式**：EPUB（PDF支持待开发）
- **解析策略**：
  - 利用EPUB自带的章节标签和导航结构
  - 优先从OPF/Toc识别章节结构
  - 通过XML解析提取HTML正文
- **技术实现**：使用epub_plus库进行EPUB解析，按需提取内容

### 3.2 AI分层阅读引擎
- **全书概览生成逻辑**：
  - **有前言/序言**：直接从前言内容生成全书摘要（800-900字，Markdown格式）
  - **无前言/序言**：等待所有章节摘要生成完成后，基于章节摘要生成全书摘要
- **章节精炼**：用户点击目录进入某一章时，AI对该章节内容进行摘要提炼（800-900字，Markdown格式）
- **输出格式**：统一使用Markdown格式，包含标题、段落、列表等，便于阅读和导出
- **技术实现**：调用智谱/通义千问API，使用集中管理的提示词模板（ai_prompts.dart）

### 3.3 用户交互与控制
- **逃逸机制**：在任何AI摘要页面提供"阅读原文"按钮，直接跳转到原文
- **智能预加载**：在用户打开书籍详情时，后台静默预生成章节摘要
- **章节导航**：仅在第一级目录间遍历，避免子章节干扰用户体验
- **技术实现**：使用Flutter的异步任务和后台处理机制，确保预加载不影响用户体验

## 四、数据存储方案

### 4.1 存储方案
- **SQLite数据库**：使用drift ORM进行类型安全的数据库操作
- **表结构**：
  - `books` - 书籍信息（id, title, author, file_path, current_chapter等）
  - `chapter_summaries` - 章节摘要
  - `book_summaries` - 全书摘要

### 4.2 文件结构
- 数据库文件存储在应用专用目录：
  - Windows: `C:\Users\{username}\AppData\Local\zhidu\zhidu.db`
  - Android: `/data/data/{package_name}/databases/zhidu.db`
  - iOS: `Documents/zhidu.db`

### 4.3 读写逻辑
- **写入逻辑**：章节摘要和全书摘要保存到SQLite数据库
- **读取逻辑**：从数据库读取摘要数据，使用flutter_html渲染Markdown
- **导出逻辑**：支持将摘要导出为Markdown格式文件

## 五、UI/UX设计方案

### 5.1 整体设计原则
- **简洁直观**：界面设计简洁明了，突出核心功能
- **层次分明**：通过视觉层次区分不同内容模块
- **交互友好**：提供直观的交互方式，降低用户学习成本

### 5.2 主要界面设计

#### 5.2.1 首页/书架界面
- **布局**：
  - 顶部：应用标题"智读"
  - 中部：书籍列表，采用卡片式布局
  - 底部：导航栏（书架、发现、设置）
- **功能元素**：
  - 书籍卡片包含：封面图、书名、作者、阅读进度
  - 添加书籍按钮（浮动操作按钮）

#### 5.2.2 书籍详情界面
- **布局**：
  - 顶部：书籍封面和基本信息
  - 中部：目录结构（可折叠）
  - 底部：全书概览（Markdown格式）
- **功能元素**：
  - 点击章节进入阅读
  - 全书概览展示区（Markdown渲染）

#### 5.2.3 章节摘要界面
- **布局**：
  - 顶部：章节标题和导航
  - 中部：AI生成的章节摘要（Markdown格式）
  - 底部：原文链接和章节导航按钮
- **功能元素**：
  - "阅读原文"按钮
  - `<` 和 `>` 导航按钮（仅在第一级章节间遍历）
  - Markdown格式的摘要内容

### 5.3 视觉设计规范
- **色彩方案**：
  - 主色调：深蓝色（#2C3E50）用于标题和重要元素
  - 辅助色：橙色（#E67E22）用于强调元素
  - 背景色：浅灰色（#F8F9FA）用于页面背景
  - 卡片色：白色（#FFFFFF）用于内容卡片
- **字体规范**：
  - 标题字体：系统默认Bold
  - 正文字体：系统默认Regular
- **图标设计**：
  - 采用Material Design风格图标

## 八、技术风险和应对措施

### 8.1 技术风险
- **EPUB解析复杂性**：不同EPUB格式可能导致解析困难
- **AI服务成本**：AI调用可能产生费用
- **用户体验一致性**：Markdown渲染在不同平台可能有差异

### 8.2 应对措施
- **EPUB解析**：使用成熟的epub_plus库，提供回退解析机制
- **AI服务**：提供AI配置选项，支持用户选择不同提供商
- **用户体验**：使用flutter_html确保Markdown渲染一致性

本技术方案详细定义了"智读——AI分层阅读器"的技术架构、核心功能实现方案、数据存储方案、UI/UX设计方案和开发计划。通过分层阅读、智能摘要、知识沉淀三大核心功能，为用户提供高效、智能的阅读体验。

产品将以用户需求为导向，采用敏捷开发模式，逐步完善功能，最终打造一款优秀的个人知识管理工具。
